# Calculator using SLY

This lab demo exercise implements a simple **calculator** using **SLY (Sly Lex-Yacc)**, a Python library for building lexers and parsers.

## 1. SLY Lexer

A **lexer** performs lexical analysis. It reads the input expression and breaks it into a sequence of **tokens**.

For example, the expression:

```text
10 + 20 * 5
```

is converted into tokens such as:

```text
NUMBER(10)  '+'  NUMBER(20)  '*'  NUMBER(5)
```

The lexer identifies tokens such as:

* `NUMBER` – integer or floating-point numbers
* `PLUS` – `+`
* `MINUS` – `-`
* `TIMES` – `*`
* `DIVIDE` – `/`

The lexer specification is defined by creating a class that inherits from `sly.Lexer`.

## 2. SLY Parser

A **parser** takes the tokens generated by the lexer and checks whether they follow the grammar of the language.

For the calculator, the parser defines grammar rules for arithmetic expressions, such as:

```text
expression : expression + term | term
term : term * factor | factor
factor : NUMBER
```

The parser also defines **operator precedence and associativity** to correctly evaluate expressions.

For example:

```text
10 + 20 * 5
```

is evaluated as:

```text
10 + (20 * 5)
```

because multiplication has higher precedence than addition.

The parser specification is defined by creating a class that inherits from `sly.Parser`.
## 3. Execution Flow

The calculator follows these steps:

```text
Input Expression
       ↓
     Lexer
       ↓
    Tokens
       ↓
     Parser
       ↓
  Evaluation
       ↓
     Result
```

This exercise provides basic practice in using **lexical analysis and parsing** with SLY, which forms the foundation for implementing compilers and interpreters.
